## Task
Repo pakai `prisma db push` tanpa migrations history; DB Supabase berisi data production. Baseline ke `prisma migrate` supaya schema evolution aman.

## Steps That Worked
1. Verifikasi `.env` punya DATABASE_URL + DIRECT_URL (jangan print nilai).
2. `mkdir -p prisma/migrations/0_init`
3. `npx prisma migrate diff --from-empty --to-schema prisma/schema.prisma --script > prisma/migrations/0_init/migration.sql`
4. `npx prisma migrate resolve --applied 0_init`
5. Verifikasi: `npx prisma migrate status` → "Database schema is up to date!"

## Pitfalls
- **Prisma 7**: flag `--to-schema-datamodel` sudah dihapus — pakai `--to-schema`.
- DILARANG `migrate reset` / `db push --force-reset` di DB production.
- Kalau `migrate status` menunjukkan drift → STOP, laporkan, jangan auto-fix.
- Jangan asumsikan field "dead code" dari audit — grep pemakaian fungsional dulu sebelum drop kolom (kasus `isVp`: audit bilang dead, ternyata full CRUD loop di 6 file).

## Verification
- `migrate status` clean, migration.sql berisi CREATE lengkap semua model, zero drift.
