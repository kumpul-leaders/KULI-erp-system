## Task
Ganti manual type-guard validation di 17 API handler (Next.js App Router) dengan Zod, tanpa mengubah kontrak API atau business logic.

## Steps That Worked
1. Baca `prisma/schema.prisma` dulu untuk enum values yang tepat; baca tiap route existing untuk shape body aktual — schema Zod match perilaku existing (optional tetap optional).
2. Satu file per entity di `src/lib/validations/` + helper `parseBody(schema, request)` yang return `{data}` atau `{error: NextResponse 400}` dengan format error konsisten dgn existing.
3. Wire ke POST/PATCH handlers saja; GET/DELETE/FormData routes dilewati.
4. Pertahankan semua permission checks, stage gates, transactions, history creation — hanya lapisan validasi input yang diganti.
5. Cross-field validation pakai `superRefine` (contoh: periodMonth 1-4 utk quarterly, 1-12 utk monthly).

## Pitfalls
- Zod strip unknown keys by default — field yang sengaja di-exclude dari schema TIDAK akan pernah sampai ke handler (kasus isVp: checkbox settings diam-diam berhenti persist). Pastikan semua field yang ditulis ke DB ada di schema.
- Route yang butuh inspeksi raw keys sebelum validasi (mis. blok field tertentu utk non-admin) tetap pakai manual parse + `safeParse`.
