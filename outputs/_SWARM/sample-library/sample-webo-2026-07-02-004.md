## Task
Bucket `pipeline-docs` ternyata PUBLIC (dokumen quotation/contract bisa diakses siapa pun). Refactor ke private bucket + server-side upload + signed URLs.

## Steps That Worked
1. Cek status bucket via REST: `GET {SUPABASE_URL}/storage/v1/bucket/{id}` dgn service role key; ubah private via `PUT` `{"public":false}`.
2. Admin client server-only: `import "server-only"` guard + throw (bukan return null) saat key absent — cegah silent fallback ke anon key.
3. Upload: browser FormData → API route (permission-gated) → service role upload → DB simpan **path** (`leads/{id}/{type}-{ts}.pdf`), bukan URL.
4. View: endpoint auth-gated return `createSignedUrl(path, 300)` (5 menit); komponen fetch signed URL on-demand sebelum `window.open()`.
5. Backward compat: `extractStoragePath()` parse path dari record lama yang menyimpan full public URL — tanpa migrasi DB.

## Pitfalls
- Anon key (`NEXT_PUBLIC_*`) tidak boleh dipakai untuk operasi storage apapun.
- Jangan simpan public/signed URL di DB — simpan path, generate URL saat dibutuhkan.
- Service role key ada di `.env.local` (bukan `.env`) — cek keduanya.

## Verification
- curl public URL → 400/403; signed URL mengandung `/object/sign/...?token=`; expired token ditolak; build clean.
